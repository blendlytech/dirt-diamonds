# check_event_graph_integrity

**Purpose:** Validates gritty-event content batches so a shipped batch can never orphan a flag, dead-end a branch, or hide an event nothing can trigger.

**What to validate (run after EVERY content batch, before it merges):**

The content contract is `docs/design/gritty_event_framework.md` ¬ß2‚Äď¬ß4; the loader (`GrittyEventJson.Parse`) already rejects malformed JSON loudly (unknown ops/types/fields, weight outside [0,1], duplicate ids, choiceless events), so this skill's job is the *graph-level* checks the loader cannot see:

1. **Load the whole Content folder together** ‚ÄĒ `Assets/Narrative/Events/Content/*.json` parse as one `GrittyEventLibrary` (cross-batch duplicate ids fail here). The fastest full check is simply `dotnet run --project Tools/GrittyEventsHarness -c Release` ‚ÄĒ its first check parses the shipped batches.
2. **No orphaned prerequisites:** every `flag_active`/`flag_inactive` flag name referenced by any prerequisite must be writable by some `set_flag` consequence in the library (or be a documented engine-written flag, e.g. `ped_active`). A prerequisite flag nothing sets = an unreachable event.
3. **No dead-end intent:** every `set_flag` consequence should be read by at least one prerequisite somewhere (a flag nothing reads is inert ‚ÄĒ allowed for forward hooks, but list them in the batch review).
4. **No unpaced loops:** an event whose every choice leaves its own prerequisites satisfied AND has `cooldown_days: 0` will fire daily forever. Require a cooldown, a self-blocking flag, or a prerequisite the consequences break.
5. **Scope sanity:** `avatar`-scoped events whose prerequisites can never hold for a rostered adult (e.g. `age < 16`) are unreachable until heirs can be avatars ‚ÄĒ flag them. **Age/tier reachability:** a `tier` gate must be satisfiable by the population it targets (HS content needs `tier==0` reachable by a fresh avatar; pro/minor-league content needs `tier>=2` reachable post-graduation) and must not silently exclude the whole population it was meant for.
6. **Weight sanity:** `scope: any` √ó high weight multiplies by ~136 subjects; anything above ~0.1/day on an `any` event needs a justification (the engine's MaxFiresPerDay=8 valve caps the blast radius but won't fix spammy content).
7. **The NULL-team tier leak class (schema v7+/HS onboarding arc):** any `scope: any` event gated on `tier` must be checked against an UNROSTERED subject (NULL `team_id` ‚ÄĒ seeded parent NPCs, a benched/displaced teammate), which polls as tier sentinel `-1`. The sentinel defeats `>=`/`==` gates over the real 0‚Äď5 range, but **not** a `<` gate (`-1 < N` holds for any `N >= 0`) ‚ÄĒ a `scope: any` event authored with `{ "field": "tier", "op": "<", ... }` can leak onto unrostered subjects. Prefer `scope: avatar` or an explicit `>=` floor for any new tier-gated content.
8. **Contact resolution:** every `contact` id a batch authors must exist in `Assets/Narrative/Contacts/contacts.json` (an omitted `contact` defaults to the reserved `unknown` thread; `ContactRegistry` synthesizes that one even if the registry drops it). The harness's contact-registry check pins `unresolved == 0` over the whole shipped folder, so a batch minting a new thread must add the contact row in the same change. The reverse is NOT required: a contact row no shipped event tags is legal and renders nothing (`getting_started` is exactly that ‚ÄĒ kept as a dormant hint-thread hook after its tutorial batch was retired for the TutorialOverlay, 2026-07-11). Shipped ids (as of the tutorial-overlay slice): `unknown`, `coach_reyes`, `agent_diaz`, `girlfriend_jess`, `family_mom`, `family_dad`, `hs_sweetheart`, `family_kid`, `coach_hs`, `hs_friend`, `getting_started`.

**Field/op/consequence vocabulary** (loader-enforced, listed here for authoring): fields `funds|age|recklessness|health_ceiling|detection_risk|baseball_interest|strictness|teammate_ex_of_partner|tier|gpa`; ops `< <= > >= == !=`; consequences `funds|stress|interest|detection_risk|health_ceiling|set_flag|clear_flag|relationship|conceive_child|absence|person_stat|end_partnership|rekindle_partnership|child_development` (relationship: `kind` rival/friend/partner, `target` teammate/opponent/league/teammate_ex_of_partner; `absence`: `{"type":"absence","reason":"injury"|"suspension"|"arrest","days":N‚Č•1}`; `person_stat`: one of the twelve `Player_Person` integer columns, GPA excluded ‚ÄĒ it moves only through the weekly closed form; `child_development`: axis `care|coaching|funding|neglect`, avatar-scope only; `end_partnership`/`rekindle_partnership`: avatar-scope only). `min_days_since` on `flag_active` is the cascade clock. `tier` values are `LeagueTier` ordinals (`HS=0, College=1, MinorA=2, MinorAA=3, MinorAAA=4, MLB=5`); `gpa` is 0.0‚Äď4.0, schema default 2.5.

**Marriage & conception notes** (`docs/design/marriage_and_conception.md`): `conceive_child` is payload-free and loader-rejected on any non-`avatar` scope. A `relationship kind=partner` consequence is silently skipped when the subject already has a Partner edge (single-partner exclusivity) ‚ÄĒ an arc that assumes the marriage landed should gate its follow-ups on a flag the same choice sets (the ¬ß5 `married`/`expecting` cascade), never on the edge itself. Conception pacing (how many kids) is content's job via flags/cooldowns; the engine allows unlimited requests.

**Roster availability notes** (`docs/design/roster_availability_triad.md`): `absence` benches the subject via `Player_Absences`/`AvailabilityLedger`; `days` semantics = `until_day = fire_day + days + 1` (games missed, not calendar days offseason). Overlapping absences don't stack ‚ÄĒ whichever ends later wins wholesale, so a shorter absence landing on an already-benched subject is a silent no-op; author cooldowns/thresholds with that in mind. Injury rust is engine-computed at apply time from live `health_ceiling` ‚ÄĒ do not also author a rating-debuff consequence for an injury absence. `narc_watchlist`, `bad_product`, `controls_turf_local`, `hot_goods`, `robbery_bust`, `spoiled_goods`, `gambling_bust` are documented engine-written flags (written by `HustleService` ‚ÄĒ Phase 8b's narcotics/turf trio plus the later robbery/fencing/Hold'em additions; corrected 2026-07-13, DIRT-3: the turf flag is `controls_turf_local`, not the stale `controls_turf_f`/`controls_turf_<n>`) ‚ÄĒ a content batch may gate on them (`flag_active`) without also writing them; check #2's "orphaned prerequisite" rule treats these as pre-satisfied, same as `ped_active`.
