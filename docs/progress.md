## 10c OFP-scale ruling: CLOSED (2026-07-06, Fable 5 — halving UPHELD as permanent)

**The disclosed 10c finding is ruled: Sonnet's `/2` in `ScoutingGrade.OfpRating` is the correct permanent recentering. `PromotionScore.Scouting`'s `/150` divisor must NOT be re-derived. The defect was in the design doc, which misstated `Scouting` as "returns a 0–100 overall"; the doc is now corrected.**

- **Why the divisor stands:** `Scouting`'s 100-centring is load-bearing, not accidental. `Combine(P, S) = wP·P + wS·S` mixes it against `BatterPerformance`/`PitcherPerformance`, which are explicitly 100-centred by construction (`100.0 * (1.0 + w·(rawP − 1.0))`); the weights only mean their stated ratio if both operands share that scale. Re-deriving `/150` → `/300` would silently halve the effective `ScoutingWeight`, re-ordering every promotion pool — **a band move in closed 9c** (conservation-law harness + pinned fixtures) and a violation of Phase 10's no-`Simulation/`-touch construction guarantee. Compensating by doubling `ScoutingWeight` would churn a closed, fixture-pinned system for zero behavioral change while *creating* the scale mismatch inside `Combine` it purported to fix.
- **Why the halving is exact, not a fudge:** at peak age with zero headroom, `Scouting = 100·sum/150 = 2 × role-average rating`, so `Scouting/2 = roleAvg + ProjectionBonus/3` — precisely the 0–100/50-average `PlayerRatingsRow` scale `Grade`'s bands are calibrated to (150→50→C+, 120→40→C, both fixture-pinned), with `ProjectionBonus` correctly converted from role-sum units to per-rating units by the same factor.
- **Why `Grade`'s domain can't absorb it instead:** the bands also grade raw per-tool ratings (`Grade(current_i)`/`Grade(potential_i)` present/future), which are natively 0–100/50-average. OFP is the *only* consumer whose input arrives in Combine-units; unit conversion at that one boundary consumer is the right altitude.
- **Recorded in four places:** `ScoutingGrade.OfpRating`'s doc comment (flag → ruling), the harness suite comment in `Tools/MonteCarloHarness/Program.cs`, the design doc (`§5.1`'s scale claim corrected, `§5.3`'s formula now `Grade(round(Scouting(...)/2))` with the ruling inline, acceptance check 4 updated), and this entry. No executable code changed — comments/docs only.
